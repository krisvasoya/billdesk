# BillDesk — Environment Variables Reference

Secrets must never be hardcoded or checked into source control. The environment variables below are divided into backend and mobile client profiles.

---

## 🛰️ 1. Backend variables (`backend/.env`)

| Variable | Description | Requirement | Example |
| :--- | :--- | :--- | :--- |
| `PORT` | Local network binding port | Optional (Default: 4000) | `4000` |
| `DATABASE_URL` | PostgreSQL connection pool URL | **Required** | `postgres://pg:pwd@neon.tech/billdesk?sslmode=require` |
| `DIRECT_URL` | Non-pooled Direct PostgreSQL connection | **Required** for migrations | `postgres://pg:pwd@neon.tech-direct/billdesk` |
| `JWT_SECRET` | Secret key used to sign Access Tokens | **Required** (Min 8 chars) | `Kz67aM29pQx109Lz` |
| `JWT_REFRESH_SECRET` | Secret key used to sign Refresh Tokens | **Required** (Min 8 chars) | `Yx02k918Lz0091Pl` |
| `UPSTASH_REDIS_REST_URL` | Upstash Redis REST endpoint | Optional (Enables cache) | `https://upstash-instance.upstash.io` |
| `UPSTASH_REDIS_REST_TOKEN` | Upstash Redis Authorization Token | Optional (Enables cache) | `AbCdeFgHiJkLmNoP12345` |
| `CLOUDINARY_CLOUD_NAME` | Cloudinary identification cloud | Optional (Enables media) | `billdesk-cloud` |
| `CLOUDINARY_API_KEY` | Cloudinary API Credentials | Optional (Enables media) | `123456789012345` |
| `CLOUDINARY_API_SECRET` | Cloudinary Encryption Secret | Optional (Enables media) | `MySecretCloudinaryKey123` |
| `GOOGLE_CLIENT_ID` | OAuth2 Credentials for Google Sign-In | Optional | `my-app-id.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | OAuth2 Credentials for Google Sign-In | Optional | `GOCSPX-secret` |

## 🛠️ How to Generate or Obtain Secrets

### 🔑 1. JWT_SECRET and JWT_REFRESH_SECRET
These are secure cryptographic keys generated by you. Do not hardcode them.
- **Generate using Node.js:**
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
- **Generate using PowerShell:**
  ```powershell
  [Convert]::ToBase64String((1..32 | % { [byte](Get-Random -Min 0 -Max 256) }))
  ```

### 🌐 2. GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET
These are obtained from the Google Cloud Platform (GCP) Console to enable Google Sign-In authentication:
1. Navigate to the **[Google Cloud Console](https://console.cloud.google.com/)**.
2. Create or select a Project.
3. Open **APIs & Services** > **OAuth consent screen**, select user type (External), and input your app name.
4. Open the **Credentials** page:
   - Click **Create Credentials** > **OAuth client ID**.
   - Select **Web application** as the Application Type.
   - Under **Authorized redirect URIs**, add your production Vercel backend routing endpoints (e.g. `https://your-app-domain.vercel.app/api/auth/callback/google`).
5. Copy the generated **Client ID** and **Client Secret** into your backend `.env` variables.
